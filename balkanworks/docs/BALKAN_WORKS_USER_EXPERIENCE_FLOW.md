# Balkan.works User Experience Flow

## Vision

**A digital assistant for my city.** Balkan.works saves time and money by making useful local information easy to discover, trust and act on.

## Journey

`Discovery → Registration → Personalization → Daily use → Trust → Habit`

## First-use flow

1. **Splash:** Balkan.works mark and the promise “Jedna aplikacija za sve oko tebe.”
2. **Location:** choose country, city and optional neighbourhood/GPS permission with a clear explanation.
3. **Interests:** food, shopping, services, sport, events and savings; every choice can be changed later.
4. **Goal:** find offers, support local businesses, save money, or another explicit purpose.
5. **Home:** immediately show useful local content or a helpful empty state — never an unexplained blank feed.

## Home and discovery

Home has a greeting, universal search (“Šta tražiš?”), nearby offers, categories, recommendations and local businesses. Personalization begins with user-selected interests and location. AI-based ranking is opt-in/permissioned when enabled; it is not required for a useful first version.

Search accepts business, product, offer and service queries. A later Balkan AI query such as “Nađi mi jeftin ručak blizu mene” must return grounded results, their source and a visible action — not an unverified answer.

## Offer and business experience

An offer presents image, title, price/discount, business, distance and validity, with clear actions to save, share, use or follow. A business page presents logo, description, location, hours, contact, offers, reviews and trust/verification status. Actions never hide price, validity or who fulfils the offer.

## Save Food journey

`Business publishes surplus → eligible user receives a useful notification → reservation → pickup → experience rating`

Reservation state, pickup deadline and cancellation path must remain visible. Notifications are relevant, user-controlled and never a substitute for clear in-app status.

## Profile and trust

Profile contains identity, location/interests, saved items, activity history, future rewards and settings for privacy, notifications and security. Users can follow businesses, categories and offers; this signal improves recommendations only under the platform data rules.

## Business journey

`Register → create profile → add first offer → see outcome → improve`

The Business Dashboard highlights actionable measures: views, clicks, reservations/customers and campaign performance — not vanity metrics.

## System states

- **Empty:** explain why no content exists and offer the next useful action.
- **Loading:** preserve context and show progress without misleading placeholders.
- **Error:** use plain language, recovery action and support path; never raw technical error codes.
- **AI:** clearly identify suggestions, sources and any action needing confirmation.

## Daily habit loop

`Fresh local value → respectful notification or discovery → useful action → visible outcome → return`

Measure activation, search success, offer actions, sessions, repeat use and retention. Optimize for useful completed outcomes, not time spent in the application.
