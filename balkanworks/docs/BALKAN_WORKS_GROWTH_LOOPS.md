# Balkan.works Growth Loops

## Principle

Build a durable growth engine, not isolated campaigns. Every loop must create measurable value for users, businesses or communities before it is scaled.

## Core loop

`More active businesses → more useful local content → more successful user searches/actions → stronger business outcomes → more active businesses`

Supply quality and offer freshness are prerequisites; registrations or downloads alone do not prove a loop.

## Growth loops

| Loop | Mechanism | Metric to validate |
| --- | --- | --- |
| Business acquisition | Existing business outcome → referral/partner/sales → new active profile | Activated businesses, time to first offer, retained supply |
| User discovery | Useful search result → solved need → share/return | Search success, activation, organic/referral source |
| Content/SEO | More profiles/offers/local stories → stronger local discovery → visitors → supply | Indexed quality pages, organic activation, freshness |
| Review/trust | Real experience → review → clearer trust → more confident actions | Verified review rate, trust/action conversion, disputes |
| Offer | Business publishes current offer → user action → visible outcome → more offers | Offer freshness, views/saves/actions, business retention |
| Referral | User shares relevant business/offer → invited user activates | Invites, activated referrals, abuse/fraud rate |
| Partnership | Trusted local partner → community access → qualified users/businesses | Partner-sourced activation, retention and cost |
| AI (later) | Permissioned activity → improved grounded ranking → better outcomes | Recommendation quality, overrides, incremental success |

## Growth metrics

- **Acquisition:** new users/businesses and attributable source.
- **Activation:** first successful discovery, first save/contact/action, first business outcome.
- **Retention:** repeat consumer usage, active business/offer continuity.
- **Referral:** shares/invites, accepted/activated invitations and quality safeguards.
- **Network health:** local supply density, search result usefulness and supply-demand balance.

## Experiment system

Each experiment has a hypothesis, target cohort/city, expected value, primary metric, guardrail metric, duration, owner and decision rule: ship, iterate or stop. Initial experiments may test local business profile onboarding, high-quality local SEO pages and useful sharing/referral behavior.

## Guardrails

Do not purchase growth that fails retention, prompt referrals with misleading rewards, create SEO pages without useful current content, manipulate reviews or expand to more cities before the first local loop works. Growth is constrained by privacy, trust, moderation and support capacity.

## Long-term engine

Local business supply, community trust, useful data and permissioned AI reinforce one another. Each new participant should improve local relevance — otherwise the platform has activity without a durable network effect.
