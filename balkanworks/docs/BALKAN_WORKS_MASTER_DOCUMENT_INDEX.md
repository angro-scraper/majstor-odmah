# Balkan.works Master Documentation Index

## Purpose

This is the central navigation index for Balkan.works strategy, product, technical and operating documentation. It explains what is being built, why, how and in what order. It links to the actual `docs/` files rather than maintaining a duplicate folder hierarchy.

## Source-of-truth hierarchy

When documents overlap, use this order:

1. [Master Document](BALKAN_WORKS_MASTER_DOCUMENT.md) — enduring project direction and principles.
2. [Master Plan](BALKAN_WORKS_MASTER_PLAN.md) — executive synthesis of the company/product plan.
3. [MVP Specification](BALKAN_WORKS_MVP_SPECIFICATION.md) and [PRD](BALKAN_WORKS_PRODUCT_REQUIREMENTS_DOCUMENT.md) — current V1 scope/acceptance direction.
4. [Master Roadmap](BALKAN_WORKS_MASTER_ROADMAP.md), [Module Roadmap](BALKAN_WORKS_MODULE_ROADMAP.md) and [Feature Backlog](BALKAN_WORKS_FEATURE_BACKLOG.md) — sequencing and implementation status.
5. Technical/API/security documents — implementation/release constraints.

If a conflict remains, record a decision in the operating/work system before changing code or scope.

## Reading order for a development agent

1. Master Document and Master Plan
2. MVP Specification and Product Requirements Document
3. System Design and Codebase Structure
4. Database Schema and API Contract
5. Security & Privacy Framework and Testing Strategy
6. Feature Backlog and Execution Roadmap
7. Design/UX documents relevant to the assigned surface
8. Module-specific AI/data/growth/business documents relevant to the work

Before coding, inspect the existing module/schema/API and this hierarchy. Do not invent parallel architecture, duplicate code or change data without a migration and documentation update.

## Project rules

1. Documentation is updated with material code, API, database, release and process changes.
2. Major decisions record problem, alternatives, evidence, decision, owner and review trigger.
3. Each new module defines user/business value, data model, API, UI, permissions, events/metrics, tests, security/privacy and operations/support impact.
4. The documentation index is updated whenever a new source-of-truth document is added or retired.
5. Legal, financial, security and market claims require the appropriate qualified review/evidence before external use.

## Final statement

Balkan.works is not a collection of applications. It is a modular digital ecosystem connecting people, businesses, data, AI and the region. This index keeps that ecosystem understandable as it grows.
