# Balkan.works First 10,000 Users Plan

## Purpose

Define an evidence-led plan to reach the first 10,000 registered users while prioritizing real local use, retention and successful connections over signup volume.

## Primary goal and definition

The directional milestone is 10,000 registered users, 2,500+ monthly active users and a stable organic/referral growth base. These are planning hypotheses to calibrate to the pilot city, supply quality and product evidence.

A user is active when they use discovery meaningfully — for example search, business/offer viewing, saving/following or a qualified contact action — and return within the defined monthly period. The North Star is **successful, trusted connections between users and local businesses**.

## Growth philosophy

Do not chase registrations. Build a local experience worth returning to: accurate current business information, useful search results, relevant offers and clear trust signals. Demand acquisition is constrained by the quality and density of active local supply.

## Target market and audiences

Begin in one city or clearly defined region. Initial segments can include local consumers searching for services/products/recommendations, families seeking reliable local needs, professionals seeking suppliers/partners and new residents seeking trusted local orientation. Segment hypotheses are validated through interviews, behavior and local content performance, not assumed globally.

## Acquisition channels

| Channel | Role | Guardrail |
| --- | --- | --- |
| SEO and useful local pages | Long-term discoverability for real local needs | Publish only accurate, current, non-duplicated pages with clear value |
| Local content | Build local relevance and trust through guides, stories and timely resources | Do not present subjective rankings as objective facts without methodology |
| Social/community | Distribute helpful local stories, businesses and offers | Optimize for qualified activation, not impressions alone |
| Referrals | Let users share a relevant profile, offer or recommendation | Incentives must be transparent, abuse-resistant and not activated before program controls exist |
| Partnerships | Reach communities through schools, universities, organizations and local media/portals | Define attribution, privacy/data boundaries and shared value |

Paid advertising can later support a proven local loop, but is not the early primary growth engine. It must be attributable, budget-capped and judged on activated/retained users rather than clicks.

## User journey and activation

`Visitor → useful local result → profile/offer view → optional registration → return visit → regular user → successful connection`

Registration should occur when it enables a clear benefit such as saving/following, personalization, contact history or notifications. The activation event is a completed useful action: first successful search, business/offer view, save, contact or eligible recommendation interaction.

## Milestone plan

### 0 to 1,000 users — manual learning

Use local groups, direct community conversation, partner businesses, invitations and high-quality local content. Observe where people fail to find value; fix supply, search and onboarding before scaling. Track acquisition source and activation for every cohort.

### 1,000 to 5,000 users — content and discovery engine

Expand high-quality landing pages, city/category guides and business stories based on actual search demand and supply. Establish a repeatable editorial/quality process, structured metadata and freshness review. Improve referral/share flows only after core discovery is useful.

### 5,000 to 10,000 users — network reinforcement

Scale the channels with retained/activated cohorts: partner distribution, appropriate referral program, repeatable content and measured campaigns. Add categories/city areas only if profile accuracy, support/moderation and local user outcomes remain strong.

## Retention strategy

Users return for current local value: newly relevant businesses/offers, useful guides, saved-business updates and carefully controlled notifications. Notification frequency, consent and preference controls are mandatory. Retention work prioritizes better results, freshness and trust over attention-grabbing messages.

## Content engine

| Cadence | Content purpose |
| --- | --- |
| Daily (when supply supports it) | Current local updates, offers or useful short stories |
| Weekly | Practical city/category guides and curated recommendations with methodology |
| Monthly | Demand/market observations or local trend content based on responsible aggregated evidence |

Every item has an owner, location/category, source/date, update/review date and a clear user next step. Content never exposes private data or makes unverified claims about businesses.

## KPI dashboard

| Dimension | Metrics |
| --- | --- |
| Acquisition | visitors, new users, attributed source, registration conversion and acquisition cost where spent |
| Activation | first search, useful result rate, profile/offer views, saves, contacts and time to first value |
| Retention | DAU/WAU/MAU where meaningful, cohort return, repeat search/action and notification opt-out |
| Referral | shares, invitations, accepted/activated referrals, quality/fraud signals and viral coefficient methodology |
| Network health | active business supply, offer/profile freshness, search success and successful local connections |

Definitions, denominators, city, dates and known limitations are shown with each metric. A rising registration count with poor activation or retention is a signal to pause acquisition scaling.

## Growth loop

`More useful local supply → more successful searches → more user actions and learning → better relevance/content → more value → more retained users`

Data contributes only when collection is permissioned, minimal and translated into measurable relevance improvements. AI/recommendation features are introduced according to the AI roadmap, not as an early replacement for quality local data.

## Milestones and gates

| Milestone | Decision evidence |
| --- | --- |
| 100 users | First clear local discovery/use feedback and support issues identified |
| 1,000 users | Activation path and content/supply quality demonstrate repeatable promise |
| 5,000 users | Cohort retention, channel attribution and support/operations can handle growth |
| 10,000 users | A stable local loop, meaningful active-user base and repeatable acquisition/retention engine |

## Final principle

People do not come to Balkan.works for the platform itself. They come because it helps them find what they need faster and with more confidence. When that problem is solved reliably, 10,000 users become an outcome of value rather than a vanity target.
