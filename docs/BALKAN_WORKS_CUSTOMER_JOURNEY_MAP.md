# Balkan.works Customer Journey Map

## Purpose

Define the user journey through Balkan.works from first awareness to trust, repeat use and referral. Every step must create clear value: the user should feel helped, not as if they are browsing a static directory.

## User types

- **New user:** discovering Balkan.works for the first time.
- **Active user:** regularly searches, explores or connects with businesses.
- **Loyal user:** trusts the experience and recommends it to others.

```text
Awareness → Discovery → Registration → First value → Habit → Trust → Loyalty → Referral
```

## Consumer journey

| Stage | User need / action | Product responsibility | Evidence of success |
| --- | --- | --- | --- |
| Awareness | “What is this; can it help?” | communicate local value in seconds via search, social, partners and campaigns | qualified visits, first meaningful click |
| Discovery | search businesses, categories or offers | fast, simple, locally relevant results without forced registration | searches, relevant profile/offer views |
| Registration | choose to create an account | ask only for needed details and preserve value-before-registration | completion and conversion rate |
| First value | find a service, offer or save a business | guide to a useful result quickly | activation rate and time to first value |
| Regular usage | return for new needs | fresh local supply, offers, saved businesses and relevant notifications | DAU/WAU/MAU and retention |
| Trust | decide whom to rely on | verified businesses, clear information, reviews and transparent policies | report rate, satisfaction and confidence signals |
| Loyalty | make Balkan.works a default local tool | dependable recommendations, personalization and useful outcomes | repeat behavior and retention cohorts |
| Referral | share a good result | make sharing simple; reward only when it preserves trust | qualified referral rate and retained referred users |

## Registration and first-value rules

Registration is a conversion step, not an obstacle. Let visitors explore enough real supply to understand the value, then ask for an account when saving, contacting, following or personalizing requires it. Do not ask for unnecessary data upfront.

The first-value event is a verifiable moment, such as finding a useful service, discovering an offer, saving a business or initiating a business contact. It should be observable in product analytics and supported by clear recovery paths if search results are weak.

## Trust and return drivers

Trust comes from verified firms, clear contact/location information, useful reviews, current offers and transparent handling of reports. Return use comes from new local value: fresh offers, relevant recommendations, saved businesses and carefully consented notifications. Personalization must use permitted, understandable preferences and behavior signals.

## Business journey

| Stage | Business outcome |
| --- | --- |
| Awareness | understands that Balkan.works can help people find the business |
| Profile creation | completes a credible, accurate profile with onboarding support |
| First visibility | receives first profile views or relevant interest |
| Activation | publishes an offer or useful update and sees activity |
| Business value | sees qualified contacts and measured outcomes |
| Premium conversion | upgrades only after visible, value-led proof |

The business success moment is: **“This platform brings me measurable value.”**

## Pain points and product responses

| Pain point | Product response |
| --- | --- |
| complex registration | progressive, low-friction onboarding |
| insufficient local coverage | focused city/category supply acquisition before demand growth |
| lack of trust | verification, review policy, moderation and clear information |
| no visible business value | timely dashboard signals and meaningful lead/outcome definitions |

## Core journey events

Track governed events including `signup_started`, `signup_completed`, `search_used`, `business_viewed`, `offer_clicked`, `favorite_added` and `contact_clicked`. Event definitions, privacy controls and retention follow the Metrics & Analytics System.

## Experience goals

1. It is easy to find what I need.
2. I can trust the information and businesses.
3. It is worth returning when a new need arises.
4. I want to recommend a useful result.

## Final vision

The best journey is not the one with the most features. It is the journey that moves a person to real local value with the least friction and the most trust.
