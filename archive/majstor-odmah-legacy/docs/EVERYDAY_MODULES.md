# Balkan Everyday — foundation modules

Balkan.works owns these consumer modules under one Balkan ID and one permission model. They are deliberately introduced as **foundations**, not as live regulated services.

| Module | Available foundation | Requires an external agreement before live operation |
| --- | --- | --- |
| Health | provider directory and appointment requests | clinic scheduling, treatment data and medical records |
| Pharmacy | pharmacy directory and reservation requests | stock/price feed, prescription and fulfillment integration |
| Bill Pay | bill reminders | licensed payment provider and each utility operator |
| Fuel | station directory | verified real-time price feeds and partner promotions |
| Parking | parking expiry reminders | city/parking operator payment integration |
| Homes | sale, rental, roommate and short-stay listings | identity/property verification and legal moderation policy |
| Gift cards | issuer/value/code data structure | partner issuance/redemption contracts |
| Pet | veterinarian, pet shop and service discovery | provider verification and availability feed |
| Family | opt-in family groups and shared rewards setting | consent flows, parental controls and wallet provider |
| Green Score | transparent Save Food estimate | partner package weight and lifecycle-emission data |

## Safety rules

- No health record, prescription, payment, utility or municipal operator data is imported by default.
- `Bill Pay` and `Parking` remain reminder-only until a regulated/contracted payment route exists.
- Green Score exposes a provisional methodology, never an unqualified environmental claim.
- All modules remain underneath the Balkan.works Core and Balkan ID; they do not create separate registrations.
